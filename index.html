La aleatoriedad y las computadoras

Las computadoras son máquinas determinstas. Cada vez que alimentamos un programa con la misma entrada obtenemos el mismo resultado. Esto vuelve a las computadoras incapaces de producir números aleatorios. Sin embargo, las utilizamos todo el tiempo para hacer cálculos que dependen de la aleatoriedad. Por ejemplo para modelos Montecarlo, para encriptar mensajes o para seleccionar muestras de conjuntos más grandes. De tal manera que la generación de números que parezcan aleatorios es muy importante.

Esta página muestra algunos experimentos con simulaciones de números aleatorios producidos por la computadora en la que la estés viendo. Contiene herramientas para generar números pseudoaleatorios, para jugar con ellos y mostrar algunas de sus propiedades. Cada que cargas la página se producen nuevas versiones de las gráficas y de los colores utilizados para explorar la aleatoriedad.

¿Qué es aleatorio?

Cuando un fenómeno no se puede predecir decimos que es aleatorio. Por lo tanto, un solo número no puede ser aleatorio. La manera en la que se produce el número podría serlo. Cuando algo es muy difícil de predecir resulta muy útil describirlo como si fuera aleatorio. Una secuencia de números, por ejemplo, la pensamos como aleatoria si los números no dependen unos de otros. Es decir, si no puedo calcular el número siguiente usando los números previamente generados.

Las computadoras son capaces de generar secuencias de números pseudoaleatorios, es decir, que se comportan como si fueran aleatorios. La correlación entre los números de una secuencia de este tipo debe ser mínima. Una de las maneras de generar estas secuencias es con el Generador Lineal Congruencial. Un algoritmo que a partir del número anterior produce el siguiente. Este algoritmo utiliza 4 parámetros.

  • el módulo, m
  • el multiplicador, a
  • el incremento, c
  • el número inicial o semilla, r0

A partir de estos cuatro parámetros calcula números pseudoaleatorios entre 0 y el valor del módulo. Este algoritmo genera secuencias de números y conforme el valor de los parámetros estos números tienen mayor o menor correlación entre ellos. Una de las desventajas del algoritmo es que produce secuencias que después de una cierta cantidad de números se repite.

Caminantes aleatorios

Se pueden hacer muchas cosas con números aleatorios. Una de las más interesantes son las caminatas aleatorias. Supongamos que un caminante se encuentra en el punto x=0 y y=0 al principio de la caminata (t=0). Este caminante va a emprender una caminata, pero la dirección de sus pasos serán decididos por números aleatorios. A cada instante cambia la dirección de su caminata de forma aleatoria y da un paso. ¿Qué tan lejos llegaría un caminante que cambia su rumbo de forma aleatoria todo el tiempo?

En la gráfica anterior se puede observar que un caminante aleatorio deambula por el espacio de forma errática como era de esperarse. Sin embargo, podemos notar también que ciertas regiones del espacio se llenan con su caminata. Cada vez que se cambia el número de pasos la caminata se dirige hacia una región diferente. Si repetimos las caminatas muchas veces se cubrirían todas las direcciónes. Sin embargo, en términos de distancia pareciera que mientras más dura la caminata el caminante es capaz de llegar más lejos. ¿Hasta dónde llega un caminante aleatorio?

Distancia promedio de un caminante

Si un caminante recorre muchas caminatas con el mismo número de pasos podríamos promediar la distancia a la que llega cada una de esas caminatas. Es decir, para un número de pasos N podemos calcular la distancia promedio a la que llega el caminante.

Analíticamente se puede calcular que esta distancia varía con el número de pasos y es proporcional a la raíz cuadrada de N. Es decir, mientras más pasos tenga la caminata más lejos llegará el caminante en promedio. ¿Qué tan lejos? Una distancia igual a la distancia de un paso multiplicada por la raíz de el número de pasos de la caminata.
